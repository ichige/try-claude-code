# Auth Store の作成

共通ライブラリとして作成した MSAL 認証機能を Router Guard として実装してみる。

## prompt

まずは boot での初期化が必要である。

```markdown
`apps/admin` 配下の Quasar APP の boot で、`packages/shared/src/auth/index.ts` から MSAuth を import して初期化、
Store で import 出来るように実装してください。

- clientId, authority, redirectUri は環境変数を参照するように設定して。
- .env も作成して、ダミー値で定義しておいて。
- 必要であれば、ImportMetaEnv にも定義をしておいて。
```

Store はまだ実装しなくても良かったけど、なぜか実装してくれた。  
まぁ、ある意味単純ではあるけど。

この Store を使って、Router Gurd で認証を実行してもらう。

```markdown
作成してもらった auth store を利用して、Router の Navigation Guards にポップアップ認証組み込んでください。
- まずは全てのページ有効にするという想定で実装してください。
```

また何故か余計な修正が入ったので、再修正を依頼する。

```markdown
- acquireToken が void となり、トークンがステート管理となっていますが、API実行時に都度最新のトークンを発行するので、元の通りトークンを返すように修正してください。
- 当然ですが、Navigation Guard で発行する必要はありません。
```

scope の参照が微妙なので、再調整する。

```markdown
- ログイン時の scope は `['User.Read']` 固定で問題ないです。
- 環境変数も不要です。
```
