# Clade Code お試し

- phpstorm + Claude Code でアプリを作る。
- ベストプラクティスと言えるようなコードを目指す。
    - コードをRAGとして提供し、AIによるコード出力の精度を高めたい。

## 主な構成

- フロントエンド
    - Quasar (Vue + Router + Pinia) による SPA
- バックエンド
    - Azure Functions
- 言語
    - Typescript
- クラウドサービス
    - Azure 

個人的には JSDoc + TS(あくまでも入力補完用)で開発する方が効率が良いと感じているが、AI と TS の親和性が高いとの噂を信じて、どこまでやってくれるかを検証しつつ進めたい。

クラウドサービスは、使い倒せばある意味安いMS365(Azure)を利用する。

- Functions
- Cosmos DB

### 教訓

- 修正をお願いすると、何かに引きずられるのか？関係ないロジックを入れてくる場合がある。
    - とりあえず保存させておき、説教して削除させるみたいな？感じで対応している。
- お前は悪くない。
    - そもそもライブラリ側が完全に壊れている場合は、AIにしろ自分にしろ中身を変更できないのでどうにもならない。
    - この場合AIはなんとかしようとして、謎の修正をでっちあげてくるけど、コードが汚されるので注意すべし。
    - MSAL のポップアップの正常系が動かないとかどうかしてないか？(はまった)
- AIが混乱しはじめるポイントを見極めて、思考を停止させるのも人間の仕事である。
    - YES マンになると、永久ループみたいな状態に入り、だいたいコードを汚染して終わる。
